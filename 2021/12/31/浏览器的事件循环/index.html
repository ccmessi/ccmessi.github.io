浏览器的事件循环

本文最后更新于：2021年12月31日 晚上

为什么js在浏览器中有事件循环的机制？

  • JS是单线程的

JS代码是一行行执行的，也就是同步执行，那么如果我们想要实现异步的操作，就可以通过event loop的概念来实现了非阻塞的机制。

  • 这里涉及到两种任务

宏任务：整体代码，setTimeout，setInterval，I/o操作 （

微任务：new Promise().then，MutationObserver() *前端的回溯

  • 为什么要引入微任务的概念，只有微任务可以吗？

宏任务：先进先出的原则执行。此时如果出现高优先级的任务，所以引入了微任务。

在任务队列中执行完了微任务，再执行下一轮宏任务。

  • Node中的事件循环和浏览器中的事件循环有什么区别？

宏任务的执行顺序：

  1. timers定时器:执行已经安排的setTimeout 和 setInterval的回调函数。
  2. pending callback 待定回调:执行延迟到下一个循环选代的 I/O 回调
  3. idle, prepare:仅系统内部使用。
  4. poll:检索新的I/O事件，执行与 I/O 相关的回调。
  5. check:执行 setImmediate() 回调涵致。
  6. close callbacks: socket.on(‘close’,() => {})

微任务和宏任务在node的执行顺序:
Node v10及以前:

  1. 执行完一个阶段中的所有任务

  2. 执行nextTick队列里的内容

  3. 执行完微任务队列的内容
